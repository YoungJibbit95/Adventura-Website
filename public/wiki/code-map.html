Wo die Systeme im Code liegen

Adventura ist in Gradle-Module getrennt. Diese Seite zeigt, welche Dateien für Content, Runtime, Netzwerk, Server-Logik und Client-UX aktuell am wichtigsten sind.

Module

Modul Aufgabe Typische Inhalte
common Geteiltes Spielmodell Registries, Blocks, Items, Crafting, World Data, Packets, Lighting, Entities.
server Autoritative Simulation Netty Transport, Auth Hook, Tick Loop, World Streaming, Inventory Validation.
client Rendering und UX LWJGL Window, Camera, HUD, Inventory, Chat, Settings, Chunk Meshes, Sprites.
launcher Startoberfläche Swing Launcher für Singleplayer, lokalen Server und Multiplayer Join.
tools Utility-Einstieg Asset- und Daten-Tasks für spätere Pipelines.

Server-validierter Gameplay-Fluss

  1. Client wählt Hotbar-Slot und sendet BlockAction, BlockInteract oder CraftRequest.
  2. Server prüft Zielblock, Reichweite, Inventar, Crafting-Rezept und Station-Regeln.
  3. Server mutiert die Welt oder lehnt ab, ohne dass der Client die Wahrheit besitzt.
  4. Akzeptierte Änderungen kommen als BlockUpdate, ChunkData oder InventorySnapshot zurück.
